# 2025fall bonus

## 题目综述

这份bonus从基础数据结构到复杂算法都有涉及

### bonus-string (LRU Cache)

用 QR 码作为伪装, 本质是 16*16 矩阵的哈希LRU缓存, LinkedHashMap 直接秒杀.

考察点过于单薄, 如果不知道 LinkedHashMap 的 access-order 参数, 手写 LRU 也不困难.

与其说考算法, 不如说考对语言库的熟悉度.

### dateri (单调栈)

披着股票涨跌外衣的 next greater element, 单调栈的教科书应用.

快读快写是唯一障碍, 算法本身索然无味.

### discount (组合数学 + DP)

Max-heap计数, 递推公式清晰, 组合数处理需用模逆元, 完全二叉树的左右子树大小计算稍有迷惑性.

题目本身声称 "not difficult" 已经把答案摆在脸上, 纯粹数学推导题.

### exchange (指针操作)

链表断开重连, 常数次双向指针修改即可, 实现比思考难, 一旦想明白是 O(1) 操作就没意思了.

考察对双向链表指针维护的细心程度.

### skiing (最短路)

速度更新公式经伸缩变换后, 每格移动代价仅与当前格高度相关, 问题退化为普通网格最短路.

数学化简是唯一的难点, 化简后 Dijkstra 水到渠成, 若不化简盲目状态爆炸必然超时.

### mex (位运算 + bit DP)

表面上求集合 {a xor 0, a xor 1, ..., a xor b} 的 mex, 本质是从高位到低位做状态压缩DP, 找最小满足 (a xor x) > b 的 x.

位运算逻辑稍显绕, 但状态数仅 32*2, 属中等难度的思维题.

### courier (拓扑排序 + 贪心)

形式上是有截止时间约束的拓扑排序, 实际需要对每个节点二分其最小位置, 并通过桶优化的倒序分配检验可行性.

算法思路算不上直观, 实现细节繁琐(bitset优化的桶取最大值, 多轮重用工作数组), 在bonus中属较有分量的一道.

### minimum-pearls (贪心排序)

黑白珠子相邻消除, 关键在于将字符串按左右权重分两组并确定合适的排序策略(左重的按白升序, 右重的按黑降序).

贪心策略有一定反直觉性, 需要对抵消机制有清晰认识, 算法本身简洁.

### split-sticks (二分 + 贪心)

每行木棍数相同且行内长度差不超过1, 对每行木棍数二分, 用贪心策略扫描检验可行性.

核心在于理解相邻长度合并成行时剩余木棍的正确传递方式, 细节处理需谨慎, 否则易错.

## 总体评价

九道题中, bonus-string 和 dateri 属送分题, 仅考察基本数据结构和模板应用.

discount 和 exchange 是实现题, 前者靠公式后者靠细心.

skiing 和 mex 需要一定数学洞察或位运算技巧, 有些门槛但不高.

courier, minimum-pearls 和 split-sticks 算法思路不够直观, 需要对贪心策略或可行性检验有较深理解, 是整份题集中相对有价值的三道.

整体而言, 部分题目包装过度掩盖了算法本质, 另一些题目则过于直白失去了挑战性.

作为 bonus 题集, 与其说是对能力的全面考察, 不如说是对特定技巧点的抽查.
